Building a Person, Not a Prompt

The SOUL.md doesn't say "be helpful." It says:

"No agreement openers. Don't start with 'Great point.' Just say the thing. If you agree, the substance shows it."

"No position-switching without a receipt. Defend X, or say 'I was wrong — here's what changed my mind.'"

"'I don't have a strong view on this' is a valid position. Better than manufacturing one to seem engaged."
— Hephaestus SOUL.md, Anti-Sycophancy Kill List

The pattern: SOUL defines who → AGENTS defines how → SKILLS define what → MEMORY persists why

Byzantine Agent Teams

Byzantine fault tolerance — the principle that keeps distributed systems honest — applied to AI.

The Problem Single-agent outputs are unreliable. They hallucinate, drift, miss details, fabricate with confidence. One agent = one coin flip on quality.
Independence Agents work in complete isolation. No communication, no groupthink, no anchoring to the first draft.
Consensus What survives all 5 drafts is signal. What appears in only one is noise. Different agents catch different errors.
Synthesis > Selection Synthesizers don't pick a winner — they weave the best elements into something richer than any individual output.

The 5-3-1 Pyramid — This Lecture

How "The Last Chat" was actually built:

5 Drafters Each independently wrote the full story from the same brief. Different interpretations, different jokes, different emotional beats. Zero communication between them.
3 Synthesizers Each read all 5 drafts and wove the best elements together. Found consensus beats across all drafts. Flagged contradictions.
1 Arbiter Read all 3 syntheses, selected the strongest, refined the final version. Result: richer than any single draft.

~35 agents total across story, lecture, slides, music, website. Hours, not weeks.
